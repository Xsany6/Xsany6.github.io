Panduan Lengkap SlowHTTPTest

Simulator Serangan Denial of Service Lapisan Aplikasi

Instalasi Cara Pakai Opsi Contoh

Instalasi

Cara menginstal slowhttptest tergantung pada sistem operasi Anda. Di sebagian besar distribusi Linux berbasis penetration testing, alat ini sudah terpasang.

Di Debian/Ubuntu/Kali Linux:

Jika belum terpasang, Anda dapat menginstalnya dengan mudah menggunakan manajer paket apt.

sudo apt-get update && sudo apt-get install slowhttptest

Dari Kode Sumber (Source Code):

Untuk sistem lain atau jika Anda menginginkan versi terbaru, Anda dapat meng-compile dari kode sumber yang tersedia di GitHub.

Kunjungi repositori resmi: github.com/shekyan/slowhttptest dan ikuti petunjuk instalasi di sana.

Cara Menjalankan Perintah

Struktur dasar untuk menjalankan perintah slowhttptest adalah sebagai berikut:

slowhttptest -[mode] [options] -u <URL target>
  • -[mode]: Menentukan jenis serangan yang akan diluncurkan. Mode yang paling umum adalah:
    • -H untuk serangan Slowloris (header lambat).
    • -B untuk serangan Slow Body (POST lambat).
    • -R untuk serangan Slow Read (Apache Range Header).
  • [options]: Opsi tambahan untuk mengontrol serangan, seperti jumlah koneksi (-c), durasi (-l), atau kecepatan koneksi (-r).
  • -u <URL target>: Ini adalah parameter wajib yang menentukan URL target yang akan diserang.

Deskripsi Alat

SlowHTTPTest adalah alat yang dapat dikonfigurasi yang mensimulasikan beberapa serangan Denial of Service lapisan aplikasi. Ini membantu menguji server Anda untuk kerentanan DoS lapisan aplikasi seperti Slowloris, Slow HTTP POST, dan serangan oleh Apache Range Header.

Alat ini mensimulasikan serangan-serangan ini dengan cara yang efisien dan lambat, membuatnya sulit dideteksi oleh mekanisme pertahanan berbasis volume.

Opsi Perintah

  • -a <range> - Setel rentang awal untuk header Range.
  • -b <bytes> - Setel ukuran untuk header Range.
  • -c <number> - Jumlah koneksi target (default: 50).
  • -d <URL> - URL host proxy.
  • -e <URL> - URL host probe proxy.
  • -f <content-type> - Jenis konten untuk header.
  • -g - Hasilkan statistik dalam format CSV dan HTML.
  • -h - Tampilkan pesan bantuan.
  • -i <seconds> - Interval pengiriman data (default: 10).
  • -k <number> - Jumlah permintaan per koneksi.
  • -l <seconds> - Durasi tes dalam detik (default: 240).
  • -m <verbs> - Daftar kata kerja untuk probe.
  • -n <seconds> - Interval antar data.
  • -o <file> - Nama file keluaran.
  • -p <seconds> - Timeout untuk probe (default: 5).
  • -r <number> - Koneksi per detik (default: 50).
  • -s <bytes> - Nilai header Content-Length.
  • -t <verb> - Kata kerja permintaan (default: GET).
  • -u <URL> - URL Target (wajib).
  • -v <level> - Tingkat verbositas (0-4).
  • -w <bytes> - Ukuran jendela TCP bawah.
  • -x <bytes> - Panjang maksimum data.
  • -y <bytes> - Ukuran jendela TCP atas.
  • -z <bytes> - Baca byte dari buffer.

Contoh Penggunaan

1. Serangan Slowloris (Header Lambat)

Perintah ini akan membuka 1000 koneksi ke target. Setiap koneksi akan mengirim header HTTP secara perlahan (setiap 10 detik) untuk membuat server terus menunggu dan menghabiskan sumber dayanya.

slowhttptest -c 1000 -H -g -o stats_slowloris -i 10 -r 200 -t GET -u https://example.com/
  • -c 1000: Membuat 1000 koneksi.
  • -H: Mode Slowloris (Header).
  • -g: Hasilkan file statistik.
  • -o stats_slowloris: Nama file output.
  • -i 10: Kirim data setiap 10 detik.

2. Serangan Slow POST (Body Lambat)

Serangan ini mengirim header yang menyatakan akan mengirim data POST dalam jumlah besar (8192 bytes), tetapi kemudian mengirim isi (body) data tersebut sangat lambat (setiap 110 detik), mengikat koneksi server.

slowhttptest -c 1000 -B -g -o stats_slowpost -i 110 -r 200 -s 8192 -t POST -u https://example.com/login
  • -c 1000: Membuat 1000 koneksi.
  • -B: Mode Slow Body (POST).
  • -i 110: Kirim data body setiap 110 detik.
  • -s 8192: Memberitahu server bahwa ukuran body adalah 8192 bytes.

3. Serangan Slow Read (Apache Range Header)

Serangan ini meminta sumber daya dari server tetapi membaca responsnya dengan sangat lambat, memaksa server untuk menjaga koneksi tetap terbuka dan menghabiskan memorinya.

slowhttptest -c 1000 -R -g -o stats_slowread -r 200 -u https://example.com/ -t HEAD -x 24 -p 3
  • -c 1000: Membuat 1000 koneksi.
  • -R: Mode Slow Read.
  • -x 24: Baca 24 byte data per interval.
  • -p 3: Tunggu 3 detik antar interval pembacaan.
